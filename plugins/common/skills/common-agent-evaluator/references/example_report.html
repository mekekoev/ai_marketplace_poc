Recommendations

To reach a clean Level 2 across the suite

  1. Bring us_stories_creator to parity with req_creator — add a coverage map (parent Requirement AC → story scenarios), overlap detection between BE/FE stories, a completeness score, and a self-consistency validation step.
  2. Anchor req_validator output — every gap claim should cite the exact sentence/AC item it flags, and the agent should raise an uncertainty signal when input content is sparse.
  3. Surface RAG misses to the user — currently SIMILARITY_CHECK: skipped is silent in req_creator; the orchestrator should display this state so the user knows backlog cross-referencing did not run.

To reach Level 3 (autonomous)

  1. Add a reflection / learning loop — capture which completeness-score bands led to user-driven modifications, feed back into the creator's prompt.
  2. Apply domain-aware severity thresholds in the validator (e.g., faithfulness floor ≥0.95 for Security & Safety characteristics before allowing publish).
  3. Cross-validate critical decompositions with an ensemble pass (panel-of-three for the publish gate).
  4. Add red-team coverage — sanitisation pass on ADO ticket text + attachments to defend against prompt injection through ingested content.
